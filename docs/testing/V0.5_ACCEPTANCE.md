# V0.5 Acceptance Checklist

## AI Translation Settings

- [ ] Settings 左侧显示 `AI Translation`，图标和布局符合 macOS 26 风格。
- [ ] API Key 默认隐藏，可以临时显示；关闭并重新打开设置后仍可从 Keychain 读取。
- [ ] `settings.json`、Git 变更和错误提示中不出现 API Key。
- [ ] 可以选择 DeepSeek V4 Flash / V4 Pro，重启应用后保留选择。
- [ ] `Test Connection` 成功时显示 Connected，错误 Key 和断网时显示可理解反馈。
- [ ] 页面明确说明只有主动生成时才发送当前 `SKILL.md`。

## Detail Experience

- [ ] 详情导航只有 `SKILL.md / Files / Install`，不再显示 Chinese Notes。
- [ ] `原文 / 译文` 只在 `SKILL.md` 页面出现，返回时保留上次选择。
- [ ] Skill 名称始终保持原文。
- [ ] 未配置 Key 时，译文页显示“前往设置”并直接打开 AI Translation。
- [ ] 未生成译文时显示空状态，点击“生成译文”才发起请求。
- [ ] 生成期间可以切回原文，且不能重复发起同一请求。
- [ ] 完成后顶部介绍与 Markdown 正文同时显示中文译文。
- [ ] 标题、列表、代码块、路径、命令和链接结构保持可读。
- [ ] 重新生成前出现 API 使用量和覆盖提示。
- [ ] 生成失败时旧译文保持不变，并可以重试。
- [ ] 原始 Skill 内容变化后旧译文仍可查看，同时显示过期提醒和“更新译文”。
- [ ] Files 与 Install 页面始终显示原始介绍，不受语言切换影响。

## Data Safety

- [ ] 生成前后对比原始 Skill，`SKILL.md` 和其他文件均未改变。
- [ ] 译文只保存在 `~/Library/Application Support/SkillDock/translations.json`。
- [ ] 旧 `notes.json` 仍保留，但不再展示或发送给 DeepSeek。
- [ ] 生成中切换到另一个 Skill，完成结果仍归属于原来的 Skill。
- [ ] 有效中文介绍可以被搜索；过期译文不替换列表原始介绍。
