# V0.1 Local Skill Library Spec

## Goal

V0.1 要做成一个可靠的本地 Skill 资产台账。

用户打开 SkillDock 后，可以看到本机 Codex / Claude Code / 主技能库里的 skills，查看内容，把有用的 skill 收进主技能库，并同步到 Codex 或 Claude Code。

## User Jobs

### Job 1: 看见本机所有 skills

用户需要知道自己电脑上已经有哪些 skills，它们分别来自哪里。

需要支持：

- 扫描 `~/AI-Skills`。
- 扫描 `~/.codex/skills`。
- 扫描 `~/.claude/skills`。
- 递归识别包含 `SKILL.md` 的文件夹。
- 标记来源：主技能库、Codex、Claude Code。
- 标记系统 skill：例如 `~/.codex/skills/.system`。

### Job 2: 看懂一个 skill

用户点击 skill 后，需要快速判断它做什么、有没有风险、是否值得安装。

详情页需要展示：

- 名称。
- 英文描述。
- 来源。
- 真实路径。
- 安装状态。
- 文件树。
- `SKILL.md` 原文。
- 是否包含 `scripts/`。
- 中文备注。

### Job 3: 收进主技能库

用户可以把 Codex / Claude / 本地文件夹中的 skill 复制到主技能库。

需要支持：

- 选择本地 skill 文件夹导入。
- 检查是否包含 `SKILL.md`。
- 导入前预览名称、描述、文件结构。
- 复制到 `~/AI-Skills`。
- 同名冲突时提供跳过、覆盖、重命名。
- 默认选择跳过。

### Job 4: 同步到工具

用户可以把主技能库中的 skill 安装到 Codex / Claude Code。

需要支持：

- 安装到 `~/.codex/skills`。
- 安装到 `~/.claude/skills`。
- 同名冲突时提供跳过、覆盖、重命名。
- 默认选择跳过。
- 安装后刷新状态。

### Job 5: 添加中文备注

用户可以给 skill 添加中文理解层，但不修改原始 `SKILL.md`。

字段：

- 中文名称。
- 中文描述。
- 中文标签。
- 适合场景。
- 风险等级。
- 风险说明。
- 使用建议。

V0.1 先手动填写，不接 AI 自动生成。

## Information Architecture

### Library

主技能库页面。

展示：

- Skill 名称。
- 中文描述。
- 标签。
- 来源。
- Codex 安装状态。
- Claude 安装状态。

常用操作：

- 查看详情。
- 安装到 Codex。
- 安装到 Claude Code。
- Reveal in Finder。
- Copy Path。

### Installed

已安装页面。

分组：

- Codex。
- Claude Code。
- System。
- Personal。

系统 skills 默认只读。

### Detail

详情页。

结构：

- Overview。
- SKILL.md。
- Files。
- Chinese Notes。
- Install。

### Settings

设置页。

配置：

- 主技能库路径。
- Codex Skills 路径。
- Claude Code Skills 路径。
- 是否显示系统 skills。
- 默认安装目标。
- 默认冲突策略。

## Data Model Draft

```json
{
  "id": "wechat-article-writer",
  "name": "Wechat Article Writer",
  "description": "Write WeChat articles in a structured style.",
  "path": "/Users/name/AI-Skills/writing/wechat-article-writer",
  "source_type": "library",
  "source_tool": "codex",

  "has_skill_md": true,
  "has_scripts": false,
  "is_system": false,
  "is_readonly": false,
  "installed_targets": {

    "claude": false

  "content_hash": "sha256-value"

```

```json
  "usage_note": "适合在开始写文章前使用。"
}
```

## Metadata Strategy

V0.1 推荐：

- 主技能库中的 skill 可以保存 `.skilldock.json`。
- Codex / Claude 已安装目录中的 skill 默认不写入 metadata。
- 系统 skill 只读展示，不写入 metadata。

原因：

- 主技能库是用户资产源头，可以跟随 Git 同步。
- 工具目录只是安装目标，不应该被 SkillDock 频繁污染。

## Safety Rules

- 不静默覆盖任何 skill。
- 系统 skill 只读展示。
- 删除操作不进入 V0.1。
- 包含 `scripts/` 的 skill 默认显示风险提示。
- 写入 Codex / Claude 目录前先检测路径是否存在和可写。

## Acceptance Criteria

- 首次打开可以设置或创建主技能库。
- 可以扫描主技能库、Codex、Claude Code。
- 可以识别所有包含 `SKILL.md` 的文件夹。
- 可以查看文件树和 `SKILL.md`。
- 可以导入本地 skill 到主技能库。
- 可以安装主技能库 skill 到 Codex / Claude Code。
- 同名冲突不会静默覆盖。
- 可以编辑并保存中文备注。
- 搜索可以命中英文名称、英文描述、中文名称、中文描述、中文标签。
- 可以 Reveal in Finder。
- 可以 Copy Path。
