# SkillDock V0.4 Multi-Agent Targets Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make SkillDock support a configurable list of Agent install targets instead of hard-coding only Codex and Claude.

**Architecture:** Introduce an `AgentTarget` model as the single source of truth for install target identity, display name, path, enabled state, logo asset name, and system-skill rules. Keep Codex and Claude as default enabled targets for backward compatibility, then let the scanner, library builder, install/uninstall flow, filter menu, detail header, and Settings read from `settings.agentTargets`. New Agents such as Grok, Gemini, OpenCode, Antigravity, and Hermes can be added as presets or user-configured targets without rewriting core UI.

**Tech Stack:** Swift 6.3, SwiftUI, Swift Package Manager, local JSON settings, existing file copy/remove operators, existing Skill scanner and hash utilities.

---

## Product Scope

### Included In V0.4.0

- Replace fixed Codex / Claude install target logic with dynamic Agent Targets.
- Preserve existing user settings by migrating `codexPath` and `claudePath`.
- Keep Codex and Claude enabled by default.
- Add suggested presets for additional Agents, initially disabled until the user sets or confirms a path.
- Make Library / Installed Agent filter dynamic.
- Make detail header Agent logos dynamic.
- Make Install Targets dynamic.
- Add Settings UI for enabling/disabling Agent targets and editing paths.
- Keep all file writes explicit and confirmed.

### Not Included In V0.4.0

- No batch install/sync button.
- No automatic background sync.
- No symlink mode.
- No automatic local/remote merge.
- No AI-generated Chinese notes.
- No private GitHub login.

### Recommended Follow-up

- V0.4.1: batch install selected Skills to one Agent.
- V0.4.2: multi-Agent sync review table.
- V0.5: AI Chinese interpretation.

    case library

extension SkillSource: Codable {
    public init(from decoder: Decoder) throws {
        let value = try decoder.singleValueContainer().decode(String.self)
        self = value == "library" ? .library : .agent(value)
    }

    public func encode(to encoder: Encoder) throws {
        var container = encoder.singleValueContainer()
        switch self {
        case .library:
            try container.encode("library")
        case .agent(let id):
            try container.encode(id)
        }
    }
}

public struct SkillInstallation: Codable, Equatable, Sendable {
    public var agentIDs: Set<String>

    public init(agentIDs: Set<String> = []) {
        self.agentIDs = agentIDs
    }

    public init(codex: Bool = false, claude: Bool = false) {
        var ids = Set<String>()
        if codex { ids.insert(AgentTargetID.codex) }
        if claude { ids.insert(AgentTargetID.claude) }
        self.agentIDs = ids
    }

    public var codex: Bool {
        get { agentIDs.contains(AgentTargetID.codex) }
        set { newValue ? agentIDs.insert(AgentTargetID.codex) : agentIDs.remove(AgentTargetID.codex) }
    }

    public var claude: Bool {
        get { agentIDs.contains(AgentTargetID.claude) }
        set { newValue ? agentIDs.insert(AgentTargetID.claude) : agentIDs.remove(AgentTargetID.claude) }
    }
}
