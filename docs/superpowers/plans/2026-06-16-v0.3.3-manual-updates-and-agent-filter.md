# SkillDock V0.3.3 Manual Updates And Agent Filter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add manual GitHub update checking for imported Skills and Agent filtering in Library / Installed.

**Architecture:** Build on the existing V0.3 remote import records instead of introducing a new sync system. Remote source metadata remains in `RemoteSourceStore`; update checks acquire the recorded repository, rescan only the recorded Skill path, compare content hashes and files, then require explicit user confirmation before replacement. Agent filtering stays in app state and reuses existing Codex / Claude installation flags.

**Tech Stack:** Swift 6.3, SwiftUI, Swift Package Manager, local JSON stores, Git clone / ZIP remote acquisition, existing Skill hashing and copy utilities.

## Task 2: Add Remote Update Check Service

**Files:**
- Create: `Sources/SkillDockCore/Models/RemoteSkillUpdate.swift`
- Create: `Sources/SkillDockCore/Remote/RemoteUpdateService.swift`
- Test: `Tests/SkillDockCoreTests/RemoteUpdateServiceTests.swift`

- [ ] **Step 1: Write failing no-update test**

Create a test where the stored `installedContentHash` equals the latest remote Skill hash.

Run:

```bash
swift test --filter RemoteUpdateServiceTests/testCheckReturnsUpToDateWhenHashesMatch
```

Expected: fail because `RemoteUpdateService` does not exist.

- [ ] **Step 2: Define update models**

Add:

```swift
public enum RemoteUpdateStatus: Equatable, Sendable {
    case upToDate
    case updateAvailable
    case localModified
}

public struct RemoteSkillUpdate: Equatable, Sendable {
    public let source: RemoteSkillSource
    public let status: RemoteUpdateStatus
    public let currentContentHash: String
    public let remoteContentHash: String
    public let currentCommit: String
    public let remoteCommit: String
    public let addedFiles: [String]
    public let modifiedFiles: [String]
    public let removedFiles: [String]
}
```

- [ ] **Step 3: Implement hash comparison**

Use the recorded repository URL, branch, path, and requested acquisition method to acquire the latest repository. Scan the recorded `repositoryRelativePath`, compute remote hash, compute current local hash, and classify:

- `upToDate`: local hash equals remote hash.
- `updateAvailable`: local hash equals installed hash and remote hash differs.
- `localModified`: local hash differs from installed hash.

- [ ] **Step 4: Verify no-update path**

Run:

```bash
swift test --filter RemoteUpdateServiceTests/testCheckReturnsUpToDateWhenHashesMatch
```

Expected: pass.
