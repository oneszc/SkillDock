# SkillDock V0.1 Local Skill Library Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a native macOS 26 SwiftUI app that scans, previews, imports, and installs local Skills while keeping Chinese understanding data completely separate from original Skill folders.

**Architecture:** Use a Swift Package with an executable SwiftUI app target, a testable `SkillDockCore` library target, and XCTest coverage. Core services own scanning, parsing, merging, notes, settings, and safe file copying; SwiftUI observes one `AppModel` and renders a Finder-style three-column interface.

**Tech Stack:** Swift 6, SwiftUI, AppKit, Foundation, CryptoKit, Swift Testing/XCTest, macOS 26

---

## Prerequisite: Install and Select Xcode

This computer currently has Swift 6.3.1 and macOS 26 Command Line Tools, but full Xcode is not installed or selected. Install the latest Xcode that supports macOS 26 before running the app.

After installation:

```bash
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
xcodebuild -version
swift --version
```

```swift
public enum ConflictStrategy: String, Codable, CaseIterable, Sendable {
    case skip
    case overwrite
    case rename
}

public enum SkillFileOperationResult: Equatable, Sendable {
    case copied(URL)
    case skipped(URL)
}

public actor SkillFileOperator {
    public func copySkill(
        from source: URL,
        to destinationRoot: URL,
        strategy: ConflictStrategy,
        isSystemSkill: Bool = false
    ) throws -> SkillFileOperationResult
}
```

Rules:

- Reject sources without `SKILL.md`.
- Reject system Skills.
- Never overwrite unless strategy is `.overwrite`.
- For rename, generate `name-copy`, `name-copy-2`, and so on.
- Copy only source folder contents; notes live elsewhere and cannot be included.
