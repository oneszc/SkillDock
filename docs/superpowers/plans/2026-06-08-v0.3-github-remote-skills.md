# SkillDock V0.3 GitHub Remote Skills Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Support installing multiple Skills from public GitHub repositories through Git Clone or ZIP, record their remote sources, and provide safe manual update checks.

**Architecture:** Add focused Core models and services for GitHub URL parsing, repository acquisition, remote Skill scanning, source persistence, batch import, and update comparison. Reuse existing `SkillScanner`, `SkillHasher`, `SkillFileOperator`, and local import rules. Add separate observable app models for remote import and remote update so `AppModel` remains the coordinator rather than owning every transient field.

**Tech Stack:** Swift 6.3, SwiftUI, Foundation `Process`, `URLSession`, JSON persistence, XCTest, macOS 26.

## Plan Self-review

- Spec coverage: public links, Clone, ZIP, automatic fallback, multiple Skills, main-library-only import, source records, manual updates, file change preview, and local modification protection all have implementation tasks.
- Scope: private repositories, login, background updates, update-all, automatic merge, and automatic agent installation remain excluded.
- Type consistency: acquisition preference, repository, candidate, source record, and update types are defined before dependent services and UI tasks.
- Test strategy: Core behavior uses TDD and local fixtures; real public repository behavior remains in manual acceptance.
