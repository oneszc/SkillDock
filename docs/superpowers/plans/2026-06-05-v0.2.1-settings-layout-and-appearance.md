# V0.2.1 Settings Layout and Appearance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the empty Skill list column from Settings and add a persistent Apple-style appearance selector.

**Architecture:** Store appearance mode in `SkillSettings` with backward-compatible decoding. Switch the root view between the existing three-column Skill browser and a Settings-only two-column layout. Keep the settings category selection local to the Settings view because General is the only category in V0.2.1.

**Tech Stack:** Swift 6.3, SwiftUI, Foundation, XCTest, macOS 26
