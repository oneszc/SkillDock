# V0.5 DeepSeek Skill Translation Design

## 1. Goal

V0.5 将详情页的手动 `Chinese Notes` 升级为 AI 译文阅读能力，让中文用户可以在不修改原始 Skill 的前提下，查看 Skill 介绍和完整 `SKILL.md` 的中文译文。

第一版只支持 DeepSeek，只处理单个 Skill，不做批量翻译和多模型供应商管理。

## 2. Confirmed Product Decisions

- 删除详情页的 `Chinese Notes` Tab。
- 详情页内容导航调整为 `SKILL.md / Files / Install`。
- `SKILL.md` 页面增加独立的 `原文 / 译文` 分段控件。
- 语言控件与内容导航位于同一行，但左右分组，不合并成同一个 Tab。
- Skill 名称永远显示原始名称，不翻译、不允许改名。
- 切换到译文时，顶部介绍文案和 `SKILL.md` 正文同时显示中文。
- 译文只读，可重新生成，不支持手动编辑。
- 译文保存在 SkillDock 应用数据目录，不修改原始 `SKILL.md`。
- Skill 内容变化后保留旧译文，但明确标记为需要更新。

## 3. Detail Page Experience

### 3.1 Layout

详情页顶部继续显示：

- 原始 Skill 名称。
- 当前语言对应的介绍文案。
- Agent 安装状态。

内容导航行：

```text
[ SKILL.md | Files | Install ]                 [ 原文 | 译文 ]
```

规则：

- `原文 / 译文` 是独立分段控件，不属于左侧内容导航。
- 语言控件只在 `SKILL.md` 页面出现。
- `Files` 和 `Install` 不受语言状态影响，进入这两个页面时顶部介绍显示原文。
- 切换到 `Files` 或 `Install` 后，再返回 `SKILL.md`，保留上一次语言选择。
- 默认打开原文，避免自动触发 API 请求。

### 3.2 No Translation Yet

用户首次进入译文时显示空状态：

- 标题：`尚未生成译文`
- 说明：`SkillDock 会将当前 SKILL.md 发送给 DeepSeek 生成中文译文。`
- 主按钮：`生成译文`

如果尚未配置 DeepSeek：

- 主按钮改为 `前往设置`
- 点击后打开 Settings 的 `AI Translation` 分类。

### 3.3 Generating

- 按钮进入加载状态并禁止重复点击。
- 原文仍可切换查看。
- 请求成功前不保存部分内容。
- 生成完成后自动显示完整译文。

### 3.4 Existing Translation

- 译文按 Markdown 样式只读展示。
- 提供 `重新生成` 操作。
- 重新生成前提示会覆盖现有译文并产生新的 API 使用量。
- 请求失败时保留原有译文，不覆盖成功版本。

### 3.5 Source Changed

译文与生成时的 Skill 内容 Hash 绑定。

当原始 Skill 内容变化：

- 旧译文仍可查看。
- 页面显示 `原内容已变化，译文可能已过期`。
- 提供明确的 `更新译文` 操作。
- 不后台自动更新，不自动消耗 API。

## 4. Translation Content

一次生成返回两个独立字段：

- `translatedDescription`：顶部介绍文案的中文译文。
- `translatedMarkdown`：完整 `SKILL.md` 正文的中文译文。

翻译规则：

- Skill 名称、命令、代码、文件路径、URL、参数名和标识符保持原文。
- Markdown 标题、列表、表格、代码块和链接结构保持不变。
- 不新增原文没有的信息，不进行风险推断或使用建议扩写。
- 输出使用 DeepSeek JSON Output，解析成功后再整体保存。
- 返回空内容、无效 JSON 或缺少字段时视为失败，不保存半成品。

列表和搜索：

- 有有效译文时，列表辅助描述优先使用 `translatedDescription`。
- 无译文或译文失效时，列表回退到原始描述。
- `translatedDescription` 参与中文搜索，避免移除 Chinese Notes 后丢失中文检索能力。
- 完整译文正文暂不加入搜索索引，避免结果噪音和额外性能成本。

## 5. Settings

Settings 左侧新增 `AI Translation` 分类，使用系统设置页结构。

第一版字段：

- Provider：固定显示 `DeepSeek`。
- API Key：安全输入，可临时显示或隐藏。
- Model：选择 `deepseek-v4-flash` 或 `deepseek-v4-pro`，默认 `deepseek-v4-flash`。
- `Test Connection`：验证密钥和模型是否可用。

保存规则：

- API Key 存入 macOS Keychain，不写入 `settings.json`、日志或 Git。
- Provider 和模型选择自动保存。
- API 地址固定使用 DeepSeek 官方 `https://api.deepseek.com`，第一版不开放自定义地址。
- 测试连接只验证配置，不生成或保存译文。

设置页说明：

```text
只有在你主动生成译文时，当前 Skill 的 SKILL.md 内容才会发送给 DeepSeek。
```

## 6. Storage And Migration

译文数据独立保存在：

```text
~/Library/Application Support/SkillDock/
```

每条译文至少记录：

- Skill 原始名称。
- Skill 来源。
- 生成时的内容 Hash。
- 中文介绍。
- 中文 Markdown。
- DeepSeek 模型名称。
- 生成时间。

迁移规则：

- 现有 `notes.json` 不删除，避免用户历史数据丢失。
- V0.5 不再展示或编辑 `Chinese Notes`。
- 旧中文备注不自动拼接进 AI 译文，也不发送给 DeepSeek。
- 新译文数据使用独立存储，后续可以单独迁移或扩展。

## 7. Error Handling

需要区分并给出可行动反馈：

- 未配置 API Key：引导前往设置。
- API Key 无效：提示检查密钥。
- 模型不可用：提示更换模型或重新测试。
- 网络不可用或超时：保留现有内容并允许重试。
- 返回格式错误或空内容：提示生成失败并允许重试。
- 内容超过模型限制：提示当前 Skill 过长；第一版不做分段翻译。
- 用户切换 Skill：当前请求结果只写入发起请求时对应的 Skill，不能误写到新选中的 Skill。

错误信息不显示完整 API Key、请求正文或服务端敏感响应。

## 8. Scope Boundaries

V0.5 包含：

- DeepSeek API 配置和连接测试。
- 单个 Skill 的介绍与完整 `SKILL.md` 翻译。
- 原文 / 译文切换。
- 本地缓存、失效提示和手动重新生成。
- 移除 Chinese Notes 入口并保留旧数据。

V0.5 不包含：

- 翻译 Skill 名称。
- 手动编辑译文。
- 批量翻译或后台自动翻译。
- OpenAI、Gemini、Ollama 等其他供应商。
- 自定义 API 地址或自定义 Prompt。
- 翻译 `references`、`scripts`、`assets` 或其他文件。
- AI 风险评级、标签、使用建议等扩写能力。

## 9. Verification

自动检查至少覆盖：

- API Key 不进入普通设置文件。
- DeepSeek 请求参数和结构化响应解析。
- Skill 名称始终保持原文。
- 介绍与 Markdown 同步切换。
- 成功生成、失败重试、重新生成和旧译文保护。
- 内容 Hash 变化后的失效状态。
- 切换 Skill 时不会写错译文。
- 旧设置和旧 `notes.json` 可继续加载。
- 中文描述搜索不回退。

人工验收重点：

- 内容导航与语言控件的层级是否清楚。
- 空状态、加载、错误和过期状态是否易懂。
- 译文 Markdown 的标题、列表、代码块和链接是否保持正确。
- 设置页是否清楚说明数据何时发送给 DeepSeek。
