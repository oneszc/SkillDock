# SkillDock V0.2 Local Import and Notes Polish Design

## Goal

V0.2 让本地 Skill 导入和中文理解维护更顺手。

本期重点解决两个日常操作问题：

1. 导入 Skill 前缺少完整预览，用户不容易判断内容、脚本风险和冲突。
2. 中文备注填写方式偏基础，标签、适用场景和保存反馈不够顺畅。

## Scope

### In Scope

- Skill 列表区域支持拖拽单个 Skill 文件夹。
- 保留工具栏“导入 Skill”按钮。
- 拖拽或选择文件夹后，先显示导入预览模态窗口。
- 导入预览展示名称、说明、路径、文件结构、文件数量、脚本风险和冲突状态。
- 同名冲突默认跳过，可主动选择替换或重命名。
- 中文备注改为单页分组表单。
- 标签和适用场景支持已有建议与自由新增。
- 中文备注停止输入约 1 秒后自动保存。
- 显示正在保存、已保存和保存失败状态。
- Skill 原始名称始终作为列表和详情页主名称。
- 中文描述只替换列表副标题，并在详情页作为辅助说明展示。
- 详情页移除重复的 Overview 页签，默认打开 `SKILL.md`。

### Out of Scope

- GitHub 远程 Skill。
- AI 自动中文解读。
- 批量导入。
- 批量同步。
- 高级搜索和筛选。
- 自动更新检测。
- symlink 模式。
- 删除 Skill。

## Import Experience

### Entry Points

导入有两个入口：

- 用户将一个文件夹拖到 Library 的 Skill 列表区域。
- 用户点击工具栏中的“导入 Skill”按钮，并选择一个文件夹。

两种入口进入同一个导入预览流程。

拖拽规则：

- 每次只接受一个文件夹。
- 文件夹必须包含 `SKILL.md`。
- 拖入多个项目、普通文件或无效文件夹时，不打开预览，并显示清楚的错误提示。
- 仅 Library 列表区域接受拖拽。Installed、System 和 Settings 不接受导入拖拽。
- 拖拽悬停时，列表区域显示系统风格的导入高亮状态。

### Import Preview Modal

用户选择有效文件夹后，打开模态预览窗口。窗口必须在用户确认前保持只读，不修改任何文件。

预览内容：

- Skill 名称。
- 英文描述。
- 原始文件夹路径。
- 文件总数。
- 相对文件结构。
- 是否包含 `scripts/` 文件夹。
- 主技能库中是否存在同名文件夹。
- 当前冲突策略。

主要操作：

- 取消。
- 导入。

如果包含 `scripts/`：

- 显示醒目的系统风险提示。
- 提醒用户建议检查脚本。
- 不增加第二次确认。
- 用户点击一次“导入”即可继续。

### Conflict Handling

同名冲突时默认策略为“跳过”。

用户可以在预览窗口主动选择：

- 跳过：不写入，关闭预览并显示结果。
- 替换：明确确认后替换现有 Skill。
- 重命名：使用系统生成的可用名称导入，例如 `skill-name-copy`。

任何情况下都不允许静默覆盖。

### Import Result

导入成功后：

- 刷新 Library 列表。
- 自动选择刚导入的 Skill。
- 关闭预览窗口。
- 显示简短成功提示。

导入失败时：

- 保留预览窗口。
- 显示可理解的失败原因。
- 不留下不完整的目标文件夹。

## Chinese Notes Experience

### Layout

中文备注保留在 Skill 详情的“Chinese Notes”页签中，使用单页分组表单。

分组：

1. 中文理解
   - 中文描述。
2. 标签与适用场景
   - 标签。
   - 适用场景。
3. 风险与建议
   - 风险等级。
   - 风险说明。
   - 使用建议。

页面保持 macOS 26 原生表单风格。所有常规图标使用 Apple SF Symbols。

### Skill Identity and Description Rules

Skill 原始名称是用户识别和记忆 Skill 的主要标志，任何中文理解数据都不能替代它。

规则：

- 列表主标题始终显示原始 Skill 名称。
- 详情页主标题始终显示原始 Skill 名称。
- 中文备注页不再提供中文名称编辑项。
- 已有 `chineseName` 数据继续保留在 `notes.json` 中，保证数据兼容，但不再展示、编辑或参与搜索。
- 列表副标题优先显示中文描述；没有中文描述时显示原始英文描述。
- 详情页顶部先显示原始英文描述，再显示中文描述。
- 没有中文描述时，详情页隐藏中文描述区域。

### Detail Page Layout

详情页采用“顶部信息区 + 下方内容标签”的结构：

顶部信息区：

- 原始 Skill 名称。
- 原始英文描述。
- 可选中文描述。
- 来源、安装状态和常用操作。

下方内容标签：

- `SKILL.md`。
- Files。
- Chinese Notes。
- Install。

移除 Overview 标签，避免与顶部信息区重复。用户选择 Skill 后默认打开 `SKILL.md`。

### Tags and Use Cases

标签和适用场景均使用“已有建议 + 自由新增”模式。

规则：

- 输入时展示当前所有中文备注中已使用的建议项。
- 用户可以选择建议，也可以创建新内容。
- 相同内容忽略大小写和首尾空格后去重。
- 已添加内容使用可移除的胶囊样式展示。
- 标签和适用场景保持两个独立建议集合。
- V0.2 不增加独立标签管理页面。

### Auto-save

用户修改任意中文备注字段后，停止输入约 1 秒自动保存。

状态：

- 未修改：不显示保存提示。
- 等待保存：显示轻量的待保存状态。
- 正在保存：显示系统进度状态。
- 已保存：显示简短的已保存状态。
- 保存失败：显示明确错误，并保留当前编辑内容以便重试。

自动保存规则：

- 快速连续输入只触发最后一次保存。
- 切换 Skill 前立即保存当前待保存内容。
- 保存数据仍写入 `~/Library/Application Support/SkillDock/notes.json`。
- 不修改原始 Skill，不改变原始 Skill 内容 hash。

## Architecture

### Core

新增一个可测试的导入预览模型与服务，负责：

- 验证拖入项目是否为有效 Skill 文件夹。
- 读取元数据。
- 生成相对文件列表和文件数量。
- 标记脚本风险。
- 检查主技能库同名冲突。
- 根据选择的冲突策略执行导入。

扩展备注存储与查询能力，负责：

- 汇总已有标签建议。
- 汇总已有适用场景建议。
- 规范化和去重输入。
- 保存完整中文备注。

### App

`AppModel` 负责：

- 当前导入预览。
- 导入预览窗口显示状态。
- 导入错误和结果。
- 当前备注草稿。
- 自动保存任务和保存状态。

SwiftUI 视图负责：

- Skill 列表拖拽高亮。
- 导入预览模态窗口。
- 单页分组备注表单。
- 标签与适用场景建议控件。
- 保存状态反馈。

## Error Handling

用户可见错误必须使用容易理解的表达：

- “这个文件夹不是有效的 Skill，未找到 SKILL.md。”
- “每次只能导入一个 Skill 文件夹。”
- “主技能库中已经存在同名 Skill。”
- “导入失败，原有 Skill 未被修改。”
- “中文备注保存失败，你的编辑内容仍保留在页面中。”

不向用户直接展示底层文件系统错误作为主要提示。详细错误可以保留用于开发日志。

## Testing

### Automated Tests

- 有效 Skill 文件夹生成正确预览。
- 普通文件、多个项目和无 `SKILL.md` 文件夹被拒绝。
- 预览过程不修改源文件夹或主技能库。
- 文件数量、文件结构和脚本风险识别正确。
- 同名冲突默认跳过。
- 替换必须显式选择。
- 重命名生成可用目标。
- 导入成功后记录可被 Library 扫描。
- 标签建议和适用场景建议分别汇总。
- 建议项规范化并去重。
- 自动保存不会修改原始 Skill hash。
- 保存失败时草稿内容不丢失。

### Manual Acceptance

- 将一个有效 Skill 文件夹拖入 Library 列表，出现预览窗口。
- 拖入无效内容时显示清楚错误。
- 包含 `scripts/` 时显示风险提示，并可直接确认导入。
- 同名冲突默认跳过，并可选择替换或重命名。
- 点击工具栏导入与拖拽进入相同预览流程。
- 中文备注页面可一次查看全部字段。
- 列表和详情页始终显示原始 Skill 名称。
- 列表副标题优先显示中文描述。
- 详情顶部同时显示英文描述和可选中文描述。
- 详情页不再显示 Overview，并默认打开 `SKILL.md`。
- 标签和适用场景可选择建议，也可自由新增。
- 停止输入约 1 秒后显示已保存。
- 切换 Skill 时待保存内容不会丢失。
- 原始 Skill 内容保持不变。

## Success Criteria

- 用户可以在导入前清楚判断一个 Skill 是否值得收进主技能库。
- 导入流程不静默覆盖、不产生不完整文件夹。
- 用户维护中文备注时无需频繁点击保存。
- 标签和适用场景逐渐形成可复用的个人词库。
- V0.2 不改变 SkillDock“永远不修改原始 Skill”的产品原则。
- 中文理解层不替代原始 Skill 名称。
