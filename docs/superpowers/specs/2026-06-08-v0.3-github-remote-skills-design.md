# SkillDock V0.3 GitHub Remote Skills Design

## Goal

V0.3 让用户通过公开 GitHub 链接发现、预览、导入和手动更新 Skills。

核心流程：

```text
粘贴 GitHub 链接
→ 获取公开仓库
→ 扫描一个或多个 Skills
→ 勾选并预览
→ 导入主技能库
→ 需要时手动检查更新
```

远程 Skill 默认只进入主技能库 `~/AI-Skills`，不会自动安装到 Codex、Claude Code 或其他智能体。

## Product Principles

- 公开仓库无需登录。
- Git Clone 和 ZIP 下载都受支持。
- 两种获取方式进入同一套扫描、预览和导入流程。
- 更新必须由用户手动触发并逐个确认。
- 更新前必须展示变更文件。
- 检测到本地修改时默认阻止更新。
- 不修改远程仓库，也不自动合并用户修改。

## Scope

### In Scope

- 支持公开 GitHub 仓库链接。
- 支持公开 GitHub 仓库内文件夹链接。
- 支持自动、Git Clone、ZIP 三种获取选择。
- 自动模式优先 Clone，失败后改用 ZIP。
- 扫描仓库内所有包含 `SKILL.md` 的文件夹。
- 文件夹链接自动定位并预选对应 Skill。
- 支持勾选多个 Skills 并一次导入。
- 安装前展示名称、描述、仓库内路径、文件结构、文件数量、脚本风险和同名冲突。
- 导入后记录仓库、分支、仓库内路径、获取方式、提交版本和内容 Hash。
- Clone 仓库保存在 SkillDock 应用数据目录。
- ZIP 临时文件在完成或取消后清理。
- 支持手动检查单个远程 Skill 更新。
- 更新前展示新增、修改和删除的文件列表。
- 本地 Skill 被修改时默认阻止更新。
- 用户可以保留本地版本，或明确确认使用远程版本替换。

### Out of Scope

- GitHub 登录。
- 私有仓库。
- GitHub Enterprise。
- 自动后台检查更新。
- 一键检查或更新全部 Skills。
- 自动合并本地和远程修改。
- 自动安装到 Codex、Claude Code 或其他智能体。
- Skill 市场、推荐列表和搜索 GitHub。
- 发布或推送 Skill 到 GitHub。
- 分支切换和高级 Git 操作。

## GitHub Link Support

### Repository Link

示例：

```text
https://github.com/owner/repository
```

行为：

- 识别 owner 和 repository。
- 获取默认分支。
- 扫描仓库内全部 Skills。
- 用户勾选一个或多个 Skills。

### Folder Link

示例：

```text
https://github.com/owner/repository/tree/main/skills/example-skill
```

行为：

- 识别 owner、repository、分支和仓库内路径。
- 扫描仓库内全部 Skills。
- 自动预选链接指向的 Skill。
- 用户仍可查看和选择仓库中的其他 Skills。

### Validation

V0.3 只接受：

- `https://github.com/...` 公开仓库链接。
- `https://www.github.com/...` 公开仓库链接。

无效链接、非 GitHub 链接、私有仓库和不存在的仓库必须显示清楚错误，不展示底层 Git 或网络错误作为主要提示。

## Acquisition Methods

用户可以选择：

- 自动。
- Git Clone。
- ZIP 下载。

### Automatic

- 首先尝试 Git Clone。
- Clone 失败时自动使用 ZIP。
- 界面明确显示最终实际使用的方式。

### Git Clone

Clone 仓库存放在：

```text
~/Library/Application Support/SkillDock/Repositories/
```

规则：

- 每个远程仓库使用稳定且唯一的本地目录。
- Clone 只用于读取、扫描和后续检查更新。
- 用户选中的 Skills 仍复制到 `~/AI-Skills`。
- 不把整个仓库直接放入主技能库。
- 不修改远程仓库内容。

### ZIP Download

规则：

- 从公开 GitHub 仓库下载目标分支 ZIP。
- 解压到 SkillDock 临时目录。
- 完成扫描、导入或取消后清理临时文件。
- 保存仓库来源和下载时提交版本，用于后续手动检查更新。

## Remote Import Experience

### Entry Point

主窗口工具栏新增“从 GitHub 添加”操作，使用 SF Symbol。

打开后展示原生 macOS Sheet，包含：

- GitHub 链接输入框。
- 获取方式选择。
- “读取仓库”主要按钮。
- 读取进度和错误反馈。

### Repository Scan

仓库获取成功后：

- 递归扫描包含 `SKILL.md` 的文件夹。
- 展示仓库名称、分支、实际获取方式和扫描结果数量。
- 使用列表展示 Skills。
- 每个 Skill 展示原始名称、英文描述、仓库内路径和脚本风险。
- 用户可以勾选一个或多个 Skills。
- 文件夹链接对应 Skill 默认勾选。

### Multi-select Preview

用户继续后进入导入确认：

- 展示已选择 Skill 数量。
- 每个 Skill 可展开查看文件结构。
- 展示文件数量和 scripts 风险。
- 展示主技能库中的同名冲突。
- 每个冲突默认跳过。
- 用户可以为每个冲突选择跳过、替换或重命名。

### Import Result

确认导入后：

- 只复制选中 Skills 到 `~/AI-Skills`。
- 保存每个 Skill 的远程来源记录。
- 刷新 Library。
- 自动选择第一个成功导入的 Skill。
- 显示成功、跳过和失败数量。
- 不自动安装到 Codex 或 Claude Code。

## Remote Source Records

远程来源记录保存在：

```text
~/Library/Application Support/SkillDock/remote-sources.json
```

每个记录包含：

- Skill 原始名称。
- 主技能库目标路径。
- GitHub 仓库 URL。
- owner。
- repository。
- 分支。
- 仓库内 Skill 相对路径。
- 请求的获取方式。
- 实际使用的获取方式。
- 安装时提交版本。
- 安装时 Skill 内容 Hash。
- 最后检查更新时间。

记录独立于原始 Skill 文件夹，不能向 Skill 内写入 SkillDock 元数据。

## Manual Update Experience

### Entry Point

有远程来源记录的 Library Skill 在详情页显示：

- GitHub 来源。
- 当前记录的提交版本。
- “检查更新”操作。

没有远程来源记录的本地 Skill 不显示更新入口。

### Check Update

用户手动点击“检查更新”后：

1. 获取远程仓库最新版本。
2. 定位记录的仓库内 Skill 路径。
3. 计算远程内容 Hash。
4. 比较安装时内容 Hash、本地主技能库当前 Hash 和远程 Hash。

结果：

- 远程未变化：显示“已是最新版本”。
- 远程有变化且本地未修改：展示变更预览。
- 远程有变化且本地已修改：默认阻止更新，并展示本地修改提示。
- 远程 Skill 已删除或路径失效：显示来源不可用，不删除本地 Skill。

### Update Preview

更新预览必须展示：

- 当前提交版本。
- 远程最新提交版本。
- 新增文件。
- 修改文件。
- 删除文件。
- scripts 风险变化。
- 本地是否已修改。

操作：

- 取消。
- 保留本地版本。
- 使用远程版本替换。

“使用远程版本替换”必须明确确认，不允许静默覆盖。

### Local Modification Rule

本地修改判断：

```text
当前主技能库内容 Hash != 安装时内容 Hash
```

检测到本地修改时：

- 默认禁用普通更新确认。
- 明确提示替换会丢失本地修改。
- 用户必须主动选择“使用远程版本替换”并再次确认。
- V0.3 不提供自动合并。

## Architecture

### GitHub URL Parser

新增纯模型与解析器，负责：

- 解析仓库链接和文件夹链接。
- 规范化仓库 URL。
- 输出 owner、repository、branch 和可选仓库内路径。
- 拒绝非 GitHub 和无效链接。

### Repository Acquisition

新增仓库获取服务，统一输出本地可扫描仓库：

- `GitCloneRepositoryProvider` 负责 Clone、Fetch 和读取提交版本。
- `GitHubZipRepositoryProvider` 负责下载、解压和清理 ZIP。
- `RemoteRepositoryService` 根据用户选择协调两种方式。

获取服务只负责仓库，不负责 Skill 导入。

### Remote Skill Scanning

复用现有 `SkillScanner`、`SkillMarkdownParser`、`SkillHasher` 和文件树能力，新增仓库扫描协调层：

- 扫描一个仓库内多个 Skills。
- 保存仓库内相对路径。
- 生成远程 Skill 候选项。
- 标记文件夹链接对应的预选项。

### Import

复用现有 `SkillFileOperator` 和冲突策略。

新增批量远程导入协调层，负责：

- 将多个远程候选项转换为导入任务。
- 为每个 Skill 使用独立冲突策略。
- 导入成功后保存远程来源记录。
- 汇总成功、跳过和失败结果。

### Update

新增远程更新服务，负责：

- 获取最新仓库。
- 定位远程 Skill。
- 比较提交与内容 Hash。
- 生成文件变更列表。
- 判断本地修改。
- 在用户确认后使用现有替换策略更新主技能库。
- 更新远程来源记录。

### App State

`AppModel` 增加：

- 远程导入 Sheet 状态。
- GitHub URL 和获取方式。
- 仓库读取状态与错误。
- 远程 Skill 候选列表和选择状态。
- 批量导入预览与结果。
- 单个 Skill 更新检查状态和更新预览。

为了避免 `AppModel.swift` 继续膨胀，V0.3 将远程导入和更新状态拆分为独立可观察模型，由 `AppModel` 持有。

## Error Handling

用户可见错误使用清晰表达：

- “请输入公开 GitHub 仓库链接。”
- “无法读取这个公开仓库，请检查链接或网络连接。”
- “这个仓库没有发现 Skills。”
- “Git Clone 不可用，已改用 ZIP 下载。”
- “选择至少一个 Skill。”
- “远程 Skill 已删除或移动，本地 Skill 未被修改。”
- “检测到本地修改，默认不会覆盖。”
- “更新失败，本地 Skill 未被修改。”

底层 Git、HTTP、解压和文件系统错误保留用于日志，不直接作为主要用户提示。

## Testing

### Automated

- 仓库链接解析正确。
- 文件夹链接解析并保留分支与路径。
- 非 GitHub 和无效链接被拒绝。
- 自动模式优先 Clone，并在失败后使用 ZIP。
- 仓库内多个 Skills 被扫描。
- 文件夹链接对应 Skill 被预选。
- 多选导入只复制选中的 Skills。
- 每个同名冲突策略独立生效。
- 远程来源记录独立保存，不修改原始 Skill。
- 检查更新能区分未变化、远程变化、本地变化和路径失效。
- 文件变更列表正确标记新增、修改和删除。
- 本地修改时默认阻止更新。
- 明确确认后远程版本可以替换本地版本。
- ZIP 临时文件被清理。

### Manual Acceptance

- 粘贴公开仓库链接并成功读取。
- 粘贴仓库内文件夹链接并自动预选对应 Skill。
- Auto、Clone 和 ZIP 三种方式行为清楚。
- 仓库包含多个 Skills 时可以多选导入。
- 导入后只出现在 Library，不自动安装到 Codex 或 Claude。
- 远程来源信息清楚可见。
- 手动检查更新和变更预览可理解。
- 本地修改时不会被静默覆盖。
- 无效链接、无 Skills 仓库和网络失败提示清楚。

## Release Boundary

V0.3 完成标准：

- 公开 GitHub 仓库无需登录即可安装。
- Clone 和 ZIP 均可使用。
- 一个仓库多个 Skills 可多选导入。
- 远程来源记录完整。
- 手动更新检查、变更预览和本地修改保护可用。
- 不自动安装到智能体。
- 不包含自动后台更新、批量更新和自动合并。
