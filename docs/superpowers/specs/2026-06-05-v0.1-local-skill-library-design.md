# SkillDock V0.1 Local Skill Library Design

## Product Goal

V0.1 做成一个可靠的本地 Skill 资产台账。

用户打开 SkillDock 后，可以查看本机已有的 Skills，理解它们的用途，把需要的 Skill 收进主技能库，并安装到 Codex 或 Claude Code。

## Confirmed Scope

V0.1 包含：

- 扫描主技能库、Codex、Claude Code。
- 查看 Skill 列表、文件结构和 `SKILL.md`。
- 从本地文件夹导入 Skill。
- 从主技能库复制安装到 Codex / Claude Code。
- 手动维护中文备注。
- 中文搜索。
- Reveal in Finder 和 Copy Path。

V0.1 不包含：

- GitHub 远程安装。
- AI 自动中文总结。
- symlink。
- 自动更新检测。
- 删除 Skill。
- 内置代码编辑器。
- Mac App Store 发布。

## Platform and Distribution

- 使用 SwiftUI 开发原生 macOS 应用。
- 最低支持 macOS 26。
- V0.1 通过 GitHub Release 或本地安装发布。
- 不考虑 Mac App Store 和沙盒发布。

## Main Interface

主界面采用 Finder 式三栏结构：

1. 左栏：一级导航。
2. 中栏：Skill 列表。
3. 右栏：Skill 详情和操作。

左栏包含：

- Library。
- Installed。
- System。
- Settings。

中栏用于快速浏览和搜索，展示：

- Skill 名称。
- 中文描述。
- 来源。
- Codex / Claude 安装状态。
- 风险提示。

右栏用于理解和操作 Skill，展示：

- 基础信息。
- 中文理解。
- `SKILL.md` 原文。
- 文件树。
- 安装状态。
- Reveal in Finder。
- Copy Path。
- 安装到 Codex / Claude。

## System Structure

### Skill Scanner

职责：

- 只读扫描指定目录。
- 递归查找包含 `SKILL.md` 的文件夹。
- 读取 `SKILL.md` frontmatter 中的名称和描述。
- 检查是否包含 `scripts/`。
- 识别 Codex 系统 Skill。
- 计算内容 hash。

扫描目录：

```text
~/AI-Skills
~/.codex/skills
~/.claude/skills
```

### Skill Library

职责：

- 合并不同目录扫描结果。
- 判断 Skill 来源。
- 判断 Codex / Claude 安装状态。
- 标记系统只读状态。
- 将中文理解数据关联到对应 Skill。
- 为列表、搜索和详情页提供统一数据。

### File Operator

职责：

- 从本地文件夹导入到主技能库。
- 从主技能库复制安装到 Codex / Claude Code。
- 写入前检查路径和权限。
- 处理同名冲突。

安全规则：

- 不静默覆盖。
- 默认冲突策略为跳过。
- 系统 Skill 不允许覆盖或修改。
- V0.1 不提供删除操作。

### Chinese Understanding Layer

中文理解层只服务于 SkillDock 使用者，帮助查看和理解 Skill。

它与原始 Skill 完全分离：

- 不修改 `SKILL.md`。
- 不在 Skill 文件夹中创建 `.skilldock.json`。
- 不修改 Skill 中的 scripts、references、assets 或其他文件。
- 导入和安装时不携带中文备注。
- 未来 AI 中文总结也遵守同一规则。

保存位置：

```text
~/Library/Application Support/SkillDock/
  settings.json
  notes.json
```

中文理解字段：

- 中文名称。
- 中文描述。
- 中文标签。
- 适合场景。
- 风险等级。
- 风险说明。
- 使用建议。
- 生成或编辑时间。

关联方式：

```text
Skill 名称 + 来源 + 内容 hash
```

内容 hash 变化时，SkillDock 应提示中文理解可能需要更新，但不自动覆盖用户备注。

## Data Flow

### Launch and Refresh

```text
启动或手动刷新
→ 扫描三个目录
→ 解析 Skill 信息并计算 hash
→ 合并来源和安装状态
→ 关联中文理解数据
→ 展示列表和详情
```

### Local Import

```text
选择本地 Skill 文件夹
→ 检查 SKILL.md
→ 展示导入预览
→ 检查主技能库冲突
→ 用户确认策略
→ 复制到主技能库
→ 重新扫描
```

### Install to Tool

```text
选择主技能库 Skill
→ 选择 Codex 或 Claude
→ 检查目标路径和冲突
→ 用户确认策略
→ 复制原始 Skill 文件夹
→ 重新扫描并更新状态
```

## Settings

V0.1 设置项：

- 主技能库路径，默认 `~/AI-Skills`。
- Codex Skills 路径，默认 `~/.codex/skills`。
- Claude Skills 路径，默认 `~/.claude/skills`。
- 是否显示系统 Skills。
- 默认安装目标。
- 默认冲突策略。

设置保存在 SkillDock 应用数据目录，不写入 Skill 文件夹。

## Error Handling

- 路径不存在：提示用户创建或重新选择。
- 无访问权限：明确提示受影响目录，不静默失败。
- 缺少 `SKILL.md`：拒绝导入并说明原因。
- `SKILL.md` 无法解析：保留文件夹名称并显示解析失败提示。
- 同名冲突：要求用户选择跳过、覆盖或重命名。
- 复制失败：保留原文件，展示失败原因。
- 中文备注读取失败：不影响原始 Skill 浏览。

## Testing Boundaries

重点测试：

- 递归扫描和 Skill 识别。
- frontmatter 解析失败的降级处理。
- 系统 Skill 只读识别。
- 多来源 Skill 合并和安装状态。
- 内容 hash 变化。
- 导入和安装冲突策略。
- 中文备注与原始 Skill 完全分离。
- 搜索命中英文和中文字段。

验收底线：

- SkillDock 的任何操作都不会意外修改原始 Skill 内容。
- 未经确认不会覆盖目标目录中的 Skill。
- 扫描或备注失败不会影响原始文件。
